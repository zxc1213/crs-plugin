# 验证检查清单

在需求管理的各个关键阶段提供系统性的验证检查清单，确保每个阶段的工作质量和完整性。

## 验证检查点

## 使用方式

### 自动触发

```bash
# 设计完成后自动触发设计验证
/req 添加新功能
  ↓
brainstorm-grill 完成
  ↓
verification-checklist --design
  ↓
生成验证检查清单
```

### 手动触发

```bash
# 设计验证
/verification-checklist --design REQ-20260513-001

# 实现验证
/verification-checklist --implementation REQ-20260513-001

# 部署验证
/verification-checklist --deployment REQ-20260513-001

# 全部验证
/verification-checklist --all REQ-20260513-001
```

### 交互式验证

```bash
# 启动交互式验证流程
/verification-checklist REQ-20260513-001

# 系统会逐项询问
[1/20] 技术栈是否成熟？

[2/20] 是否有现成方案可复用？
  > 找到1个开源库，但需要定制
  > 状态：⚠️ 部分满足
  > 备注：需要评估定制成本

...
```

```markdown
- 部分通过: 1 (5%)
- 未通过: 1 (5%)
- 总体状态: ✅ 通过（有轻微问题）

## 未通过项

### 1. 性能要求是否可达

- **状态**: ❌ 未通过
- **原因**: 响应时间实测为 350ms，超过设计指标 200ms
- **建议**: 优化数据库查询，添加缓存层
- **责任人**: 开发团队
- **截止时间**: 2026-05-15

## 部分通过项

### 1. 是否有现成方案可复用

- **状态**: ⚠️ 部分满足
- **说明**: 找到1个开源库，但需要定制
- **备注**: 需要评估定制成本，预计2天

- 性能未达标可能影响用户体验

1. 优先处理性能问题
2. 评估开源库定制的性价比
3. 通过性能测试后再上线
```

## 验证规则

### 自动检查

```javascript
// 可自动执行的检查项
const autoChecks = {
  // 代码风格
  codeStyle: {
    command: 'npm run lint',
    expected: 0, // exit code
    message: '代码风格检查',
  },

  // 单元测试覆盖率
  testCoverage: {
    command: 'npm run test:coverage',
    parser: (output) => {
      const match = output.match(/Lines\s+:\s+(\d+\.\d+)%/);
      return match && parseFloat(match[1]) > 80;
    },
    message: '单元测试覆盖率 >80%',
  },

  // 类型检查
  typeCheck: {
    command: 'npm run type-check',
    expected: 0,
    message: 'TypeScript 类型检查',
  },
};
```

### 手动确认

对于无法自动检查的项目，提供详细的确认指导：

```markdown
## 检查项: 架构完整性验证

### 如何验证

1. 绘制架构图，包含所有组件
2. 标注组件间的接口和数据流
3. 检查是否有遗漏的组件
4. 确认接口定义是否完整

### 验证标准

- [ ] 所有组件都在架构图中
- [ ] 每个组件的职责清晰
- [ ] 组件间接口明确
- [ ] 数据流完整无断点
- [ ] 考虑了扩展性

### 输出

- 架构图
- 接口文档
- 数据流图
```

## 集成到工作流

### 与 quality-gates 集成

```javascript
// quality-gates skill 中集成验证检查清单
async function checkGate(gateName, reqId) {
  // 1. 自动检查
  const autoResult = await runAutoChecks(gateName);

  // 2. 生成验证检查清单
  const checklist = await generateChecklist(gateName);

  // 3. 等待手动确认
  const manualResult = await waitForManualConfirmation(checklist);

  // 4. 生成验证报告
  const report = generateReport({
    auto: autoResult,
    manual: manualResult,
  });

  return report;
}
```

## NEVER 清单

- **绝不**跳过关键验证项
- **绝不**在未通过验证的情况下继续
- **绝不**伪造验证结果
- **绝不**忽略验证报告中的风险项
- **绝不**在验证未完成时部署

## 集成说明

**触发时机**：

- brainstorm-grill 完成后（设计验证）
- 代码实现完成后（实现验证）
- 准备部署前（部署验证）
- 用户手动调用 `/verification-checklist` 时

**相关 skills**：

- `quality-gates`: 质量门禁检查（与验证检查清单互补）
- `brainstorm-grill`: 设计完成后触发设计验证
- `executing-plans`: 实现完成后触发实现验证

**输出**：

- 验证检查清单（Markdown）
- 验证报告（含通过率、风险项、建议）
- 验证状态（通过/未通过/部分通过）
