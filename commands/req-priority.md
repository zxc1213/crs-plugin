# 需求优先级评估命令

对需求进行科学的多维度评估，帮助团队合理排定优先级。

## 用法

\`\`\`bash
/req:priority <req-id>
\`\`\`

## 选项

- \`req-id\` - 需求 ID（如：REQ-20260514-001）
- \`--reassess\` - 重新评估
- \`--compare <ids>\` - 对比多个需求

## 评估维度

### 1. 业务价值 - 权重 40%

### 2. 紧急程度 - 权重 30%

### 3. 依赖关系 - 权重 15%

### 4. 实现成本 - 权重 10%

### 5. 风险评估 - 权重 5%

## 优先级等级

| 评分 | 等级 | 说明     |
| ---- | ---- | -------- |
| ≥8.5 | P0   | 立即处理 |
| ≥7.0 | P1   | 高优先级 |
| ≥5.0 | P2   | 中优先级 |
| ≥3.0 | P3   | 低优先级 |
| <3.0 | P4   | 可选     |
