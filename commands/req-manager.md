# 需求管理器命令

需求管理的**统一入口**，自动识别你的意图并路由到最合适的处理流程。

## 简单来说

你只需要说**"我要做什么"**，系统会自动：

1. 理解你的意图（创建？修改？查询？）
2. 选择最合适的工具
3. 执行相应的操作

**不需要**你手动选择命令或参数。

## 用法

```bash
# 最简单的用法 - 直接描述你的需求
/req 添加用户登录功能
/req 登录页面崩溃了
/req 查看所有需求
/req FEAT-20260514-001 怎么样了？
```

## 它能做什么？

### 🎯 智能意图识别

系统能自动识别：

| 你说的话           | 系统理解              |
| ------------------ | --------------------- |
| "添加用户登录功能" | 创建新需求（feature） |
| "登录页面崩溃了"   | 报告Bug（bug）        |
| "这个设计合理吗？" | 分析问题（question）  |
| "重构登录模块"     | 重构需求（refactor）  |
| "查看所有需求"     | 查询需求              |
| "FEAT-001的进度"   | 查询特定需求          |

### 🧠 自动推断

系统会自动推断：

1. **需求类型**：feature / bug / question / refactor
2. **处理模式**：快速分析 / 深度分析 / 自动执行
3. **优先处理流程**：应该先做什么

### 🚀 智能路由

根据你的意图，系统会自动调用：

| 你的需求   | 系统调用                     |
| ---------- | ---------------------------- |
| 创建新需求 | → req-brainstorm（深度分析） |
| 评估优先级 | → req-priority（优先级评估） |
| 检查质量   | → req-quality（质量检查）    |
| 生成测试   | → req-test-plan（测试计划）  |

## 什么时候用？

### ✅ 应该使用

- **创建需求**：所有新需求都通过 `/req` 创建
- **查询需求**：查看需求状态、列表、统计
- **不确定用哪个命令**：让系统自动选择

### ❌ 不需要使用

- **明确知道用哪个命令**：直接用具体命令更快
  - 例如：明确要做质量检查 → 直接 `/req:quality`
- **更新需求**：用 `/req:update`
- **初始化项目**：用 `/req:init`

## 与其他命令的关系

```
                    你输入: "/req 添加登录功能"
                           ↓
              req-manager (智能路由)
                           ↓
        ┌──────────┬──────────┬──────────┐
        ↓          ↓          ↓          ↓
   req-brainstorm  req-priority  req-quality  req-test-plan
   (深度分析)     (优先级)      (质量检查)   (测试计划)
```

**req-manager 是"总调度员"**，它不直接做事，而是：

1. 理解你的意图
2. 选择最合适的工具
3. 协调各个工具完成工作

## 手动使用场景

通常你不需要手动调用 req-manager，因为 `/req` 命令会自动触发。

但在以下场景可能需要手动使用：

```bash
# 重新路由需求（系统理解错了）
/req-manager --reanalyze FEAT-20260514-001

# 查看路由决策（为什么走这个流程？）
/req-manager --explain FEAT-20260514-001

# 调试意图识别（为什么理解错了？）
/req-manager --debug "添加用户登录"
```

## 重要提示

**DO:**

- ✅ 使用 `/req` 作为主要入口（让系统自动选择）
- ✅ 清晰描述你的需求（帮助系统理解）
- ✅ 相信系统的智能路由（通常是最优选择）

**DON'T:**

- ❌ 手动指定每个参数（让系统自动推断）
- ❌ 绕过 req-manager 直接调用底层命令（除非你明确知道要做什么）
- ❌ 过度干预路由决策（系统通常比你更清楚）
