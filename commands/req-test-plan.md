# 测试计划生成命令

基于设计文档自动生成完整的测试计划、测试用例和测试文档。

## 用法

\`\`\`bash
/req:test-plan <req-id>
\`\`\`

## 选项

- \`req-id\` - 需求 ID（如：REQ-20260514-001）
- \`--type <类型>\` - 指定测试类型
- \`--detail\` - 生成详细测试用例
- \`--append\` - 追加到现有测试计划

## 测试类型覆盖

### 1. 功能测试

### 2. API 测试

### 3. UI 测试

### 4. 性能测试

### 5. 安全测试

## 输出文件

\`\`\`
.requirements/<type>/REQ-XXX/
├── spec.md # 主文档（已有）
└── test-cases.md # 测试用例（生成/更新）
\`\`\`

## 相关命令

- \`/req\` - 创建需求（自动触发测试计划生成）
- \`/req:quality\` --gate 2 - 测试质量检查
- \`/req:verify\` - 执行验证检查
